📋 MANUAL: Step-by-step (if you want to run individual steps)
──────────────────────────────────────────────────────────────

Step 0: Setup (do this once)
  cd /Users/ignite/College/6TH\ SEM/bda\ lab/EL
  source venv/bin/activate
  export JAVA_HOME=/opt/homebrew/opt/openjdk@17
  export PYSPARK_PYTHON="$PWD/venv/bin/python3"
  export PYSPARK_DRIVER_PYTHON="$PWD/venv/bin/python3"
  mkdir -p /tmp/spark-events data/input model/artifacts plots

Step 1: Generate data (5-10 min)
  python -m workload.generator --n-runs 40
  
Step 2: Train ML model (1-2 min)
  python -m model.train
  
Step 3: Run benchmarks (15-30 min) [Pick ONE:]
  # All three schedulers
  python -m benchmark.runner --mode all
  
  # Or individual
  python -m benchmark.runner --mode fifo
  python -m benchmark.runner --mode adaptive
  python -m benchmark.runner --mode pasta
  
Step 4: Generate plots (1 min)
  python -m benchmark.report
  
Step 5: View results
  cat eval.txt              # Model accuracy metrics
  open plots/               # View visualizations (macOS)
  ls -la data/benchmark_*   # Check JSON files

🐛 COMMON ISSUES & FIXES
──────────────────────

Issue: "PYTHON_VERSION_MISMATCH" error
  Fix: Make sure PYSPARK_PYTHON and PYSPARK_DRIVER_PYTHON are set
  $ export PYSPARK_PYTHON="$PWD/venv/bin/python3"
  $ export PYSPARK_DRIVER_PYTHON="$PWD/venv/bin/python3"

Issue: "StackOverflowError" in PageRank
  Fix: Already fixed in code (checkpointing every 20 iterations)
       If still occurs, reduce iteration counts in benchmark/runner.py

Issue: "OutOfMemoryError"
  Fix: Reduce input file sizes in benchmark/runner.py
       Use smaller text files (500MB instead of 1500MB)

Issue: Plots not generated
  Fix: Check if benchmark JSONs exist:
       $ ls -la data/benchmark_*.json
       If missing, run benchmarks:
       $ python -m benchmark.runner --mode all
